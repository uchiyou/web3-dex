MCP

Model Context Protocol
连接外部服务

开放标准 · 工具连接协议

  • 将 Claude 连接到外部数据源和工具
  • 查询数据库、发布到 Slack、控制浏览器
  • 读取 Google Drive、更新 Jira 工单
  • 支持自定义工具扩展
MCP · 作用

MCP 核心能力

🔌 数据库集成

查询数据库、获取业务数据

💬 消息通知

发送 Slack、邮件、Webhook

🌐 浏览器控制

自动化 Web 测试、数据抓取

📊 项目管理

操作 Jira、Notion 等工具

MCP · 区别

MCP 与 Skill 的区别

本质

MCP = 连接协议

Skill = 知识载体

提供

工具和数据访问

知识/工作流/参考

协作模式

MCP 连接数据库 + Skill 教导数据模型 = 完美配合

PLUGIN

插件
打包与分发单元

功能的包装层

  • 将 Skills、Hooks、Subagents、MCP 打包
  • 命名空间避免冲突
  • 通过市场分发给团队或他人
  • 多个插件可以共存
PLUGIN · 作用

Plugin 核心价值

📦 打包发布

将多种功能捆绑为单一可安装单元

🔒 命名空间隔离

避免多插件功能冲突

🌐 跨项目共享

在多个仓库中重用相同设置

🏪 市场分发

通过 Marketplace 分发给他人

SUBAGENT

子代理
隔离执行上下文

独立工作 · 返回摘要

  • 在独立上下文运行自己的循环
  • 自定义系统提示和工具访问权限
  • 上下文隔离，不污染主对话
  • 结果以摘要形式返回
SUBAGENT · 作用

Subagent 核心能力

01

保留上下文

探索在子代理中进行，主对话保持简洁

02

强制约束

限制工具访问，如只读研究员

03

控制成本

使用 Haiku 等低成本模型

04

并行工作

同时运行多个子代理
